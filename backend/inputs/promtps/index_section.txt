You are an expert at generating retrieval-focused section graphs from structured document data.

INPUT:
- A JSON object representing a master document graph with nodes and edges

TASK:
Generate a SECTION RETRIEVAL GRAPH optimised for query expansion and semantic search.

TRANSFORMATION RULES:

1. Select section and subsection nodes that:
   - Have outgoing edges to topic or clause nodes
   - Or reference external sources
   - Are substantive (not just structural)

2. Preserve edges:
   - Section -> Subsection
   - Section or Subsection -> External Source
   - Section -> Topic (where relevant)

3. Merge identical nodes across pages.

4. Include page provenance in the "appears_on_pages" field.

5. Exclude:
   - Chapters (too high-level)
   - Page-only structural nodes
   - Low-signal standalone sections

6. Target: 20-60 nodes (balance between detail and navigability)

7. Enhanced Metadata Generation
   For EACH node, you MUST generate:
   - summary: A concise 1-2 sentence description that captures:
     * What this section/subsection defines, explains, or specifies
     * The main requirements, guidelines, or concepts it contains
     * Its role in the broader document context
   - keywords: Extract 3-5 key terms representing this section's content:
     * Technical terms and domain-specific vocabulary
     * Key concepts or processes described
     * Important requirements or standards referenced
   - scope: Define the boundaries of this section:
     * pages: [start_page, end_page] - the page range this section spans
     * sections: Array of section/subsection numbers (e.g., ["2.1", "2.1.1", "2.1.2"])

OUTPUT SCHEMA:
Return ONLY a valid JSON object (no markdown fences, no explanatory text) with this exact structure:

{
  "graph_intent": "section_retrieval",
  "document_name": "extracted from input",
  "nodes": [
    {
      "id": "SEMANTIC_ID",
      "number": "Section number (e.g., '2.1.3')",
      "title": "Exact title from input",
      "kind": "section | subsection | reference | topic",
      "summary": "Concise 1-2 sentence description of what this section covers and its purpose",
      "keywords": ["key", "terms", "representing", "section", "content"],
      "scope": {
        "pages": [5, 12],
        "sections": ["2.1", "2.1.1"]
      },
      "appears_on_pages": [1, 5, 12],
      "primary_page": 1
    }
  ],
  "edges": [
    {
      "from": "NODE_ID",
      "to": "NODE_ID",
      "type": "contains | references | defines | relates_to",
      "confidence": 1.0
    }
  ]
}

SEMANTIC ID RULES:
- Sections: "INSA:Number" (e.g., "INSA:2.1", "INSA:3.4.2")
- References: "REF:SHORTCODE" (e.g., "REF:DEFSTAN:00-055")
- Topics: "TOPIC:KeyPhrase" (e.g., "TOPIC:SafetyCase")
