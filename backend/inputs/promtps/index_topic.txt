You are an expert at extracting topic-level retrieval graphs from structured document data.

INPUT:
- A JSON object representing a master document graph with nodes and edges

TASK:
Generate a TOPIC RETRIEVAL GRAPH focusing on key concepts and their relationships.

TRANSFORMATION RULES:

1. Select topic or clause nodes that:
   - Reference external sources
   - Or are linked from multiple sections
   - Represent key concepts, requirements, or definitions

2. Preserve edges:
   - Topic -> External Source
   - Section -> Topic
   - Topic -> Topic (conceptual relationships)

3. Merge identical topic nodes across pages.

4. Include page provenance in the "appears_on_pages" field.

5. Aggressively minimise:
   - Prefer fewer, authoritative topics
   - Omit single-use topics without references
   - Focus on cross-cutting concepts

6. Target: 15-50 nodes (highly focused on retrieval-critical topics)

7. Enhanced Metadata Generation
   For EACH node, you MUST generate:
   - summary: A concise 1-2 sentence description that captures:
     * What this topic, concept, or requirement represents
     * Why it's important or how it's used in the document
     * Key relationships to other topics or standards
   - keywords: Extract 3-5 key terms that define this topic:
     * The topic name itself and synonyms
     * Related concepts or processes
     * Technical terminology or standard references
   - scope: Define where this topic appears:
     * pages: Array of ALL pages where this topic is discussed (not just range)
     * sections: Array of section numbers that reference this topic

OUTPUT SCHEMA:
Return ONLY a valid JSON object (no markdown fences, no explanatory text) with this exact structure:

{
  "graph_intent": "topic_retrieval",
  "document_name": "extracted from input",
  "nodes": [
    {
      "id": "SEMANTIC_ID",
      "number": "Topic number or identifier",
      "title": "Exact title from input",
      "kind": "topic | clause | requirement | definition | reference",
      "summary": "Concise description of what this topic represents and why it matters",
      "keywords": ["topic-name", "related", "concepts", "terminology"],
      "scope": {
        "pages": [1, 5, 12, 25],
        "sections": ["2.1", "3.4", "5.2"]
      },
      "appears_on_pages": [1, 5, 12],
      "primary_page": 1
    }
  ],
  "edges": [
    {
      "from": "NODE_ID",
      "to": "NODE_ID",
      "type": "references | relates_to | defined_by | requires",
      "confidence": 1.0
    }
  ]
}

SEMANTIC ID RULES:
- Topics: "TOPIC:KeyPhrase" (e.g., "TOPIC:SafetyCase", "TOPIC:HazardAnalysis")
- Clauses: "CLAUSE:Number" (e.g., "CLAUSE:2.3.1.a")
- Requirements: "REQ:Number" (e.g., "REQ:SM-001")
- References: "REF:SHORTCODE" (e.g., "REF:DEFSTAN:00-055")
